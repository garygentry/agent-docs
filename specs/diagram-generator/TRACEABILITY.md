# Diagram Generator — Requirement Traceability Matrix

Maps every PRD requirement (REQ-XXX-NN) to the implementation spec document(s) and
section(s) that implement it. Generated by `forge-3-specs`. Source: `PRD.md` (v2),
`tech-spec.md` (v2), spec suite `00`–`08`.

> **External / host-repo requirement references.** A few non-PRD requirement IDs are
> cited in the spec suite as external references and are deliberately **not** rows in
> the matrix below:
> - `REQ-DIAG-02` — a **`doc-site-plugin`** (downstream consumer) requirement,
>   satisfied by freezing the §5 CLI contract (`06-integration-and-packaging.md` §6).
> - `REQ-TOOLS-*` / `REQ-DISC-*` — the **host `agent-docs` emitter's** own requirement
>   vocabulary, appearing in `06` §2 (verbatim `src/model.ts` code comment) and `02`
>   §5 (rationale for the sibling schema generator). They are the host emitter's IDs,
>   not diagram-generator PRD requirements.

## Coverage summary

All 24 `diagram-generator` PRD requirements (functional + non-functional) trace to at
least one spec section. No diagram-generator PRD requirement is orphaned; every spec
document opens with a `## Requirement Coverage` table tying its sections back to
REQ/OTQ IDs. Any non-PRD IDs cited in the specs (`REQ-DIAG-02`, `REQ-TOOLS-*`,
`REQ-DISC-*`) are external host-repo / consumer references, listed in the note above —
not diagram-generator requirements.
